1. В папке `./docs/prompts` лежать все `prompts`, которые использовались для создания этого проекта, по сути это реестр задач
2. ВАЖНО! Каждый prompt который тебе поступает на выполнение сохраняй в папке `./docs/prompts` отдельным файлом в формате markdown. Название файла должно быть в формате: `YYYYMMDD-HHMMSS-<task>.md`, где `<task>` это короткая строка описывающая суть задачи, писать нужно на английском в повелительном наклонении, разделитель между слов `-`, пример: `fix-non-closing-context-menu`. Внутри файла должен быть только prompt от пользователя, без дополнительной информации.
3. После внесения любый изменений проверяй:
   - Собирается ли приложение: `make dev` (фронт+бэк+база)
   - Проходят ли юнит тесты: `make test`
   - Проходят ли e2e тесты: `make e2e`
   - После того как ты закончил не нужно останавливать приложение
4. Используй playwright mcp если тебе нужно проверить что-то в браузере
5. Когда пишешь e2e тесты:
   - ссылайся на нужные элементы по `data-testid` атрибутам, если каких-то атрибутов не хватает, добавь их к нужным элементам
   - избегай в тестах безусловных ожиданий, вроде `await page.waitForTimeout(500)`, вместо этого жди с разумным таймаутом появления на странице нужных элементов
   - все данные, которые нужно создавать в рамках теста нужно создавать в команде `E2E`, а после завершения теста их нужно удалять, чтобы тест не оставлял после себя мусорные данные
